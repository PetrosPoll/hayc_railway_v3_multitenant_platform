## Commands

```bash
# Development
npm run dev          # Start dev server (Express + Vite HMR on port 5000)
npm run build        # Production build (React → dist/public/)
npm start            # Serve production build

# Type checking
npm run check        # Run tsc across all packages

# Database
npm run db:push      # Sync schema to DB (no migration file, fast)
npm run db:generate  # Generate migration SQL from schema diff
npm run db:migrate   # Apply pending migrations

# Testing
npm run test:run     # Unit tests (Vitest, CI mode)
npm test             # Unit tests (watch mode)
npm run test:e2e     # End-to-end tests (Playwright, Chromium)
```

## Architecture

Full-stack monorepo — React frontend, Express backend, and a shared package — compiled and served as a single unit on port 5000.

```
client/src/       React 18 + Vite + Tailwind + React Router v7 + TanStack Query
server/           Express REST API + Passport.js sessions
shared/           schema.ts (Drizzle ORM table defs) + utils.ts shared by both
migrations/       SQL files generated by Drizzle Kit
```

**Request flow**: Browser → Express → routes.ts handler → storage.ts abstraction → Drizzle ORM → PostgreSQL (Neon serverless)

### Key files

| File | Role |
|------|------|
| `server/routes.ts` | All API route handlers (~25k lines) |
| `server/storage.ts` | Database query abstraction layer |
| `shared/schema.ts` | Single source of truth for all DB tables and types |
| `server/auth.ts` | Passport.js local strategy, scrypt password hashing, session config |
| `server/email-service.ts` | AWS SES / SMTP email dispatch |
| `server/scheduler.ts` | Background job queue (campaign sending, etc.) |
| `server/subscription-lifecycle.ts` | Stripe subscription state machine |
| `client/src/App.tsx` | Client-side routing, protected route wrappers |

### Path aliases

```typescript
@/*         → client/src/
@shared/*   → shared/
```

## Multi-Tenancy

Each authenticated user can own multiple **websites** (`websiteProgress` table). The `websiteProgressId` is the primary isolation key — almost every data table (`contacts`, `emailTemplates`, `newsletterCampaigns`, `transactions`, etc.) is scoped to it.

Always filter by `websiteProgressId` (or `userId`) when querying tenant-scoped data. Never join across tenants.

## Database

- ORM: **Drizzle ORM** — all table definitions live in `shared/schema.ts`
- Add a table: edit `shared/schema.ts` → `npm run db:generate` → `npm run db:push`
- Migrations auto-run on server startup via `runMigrations()` in `server/index.ts`
- Session table is managed by `connect-pg-simple` (auto-created)

## Authentication & Roles

Sessions stored in PostgreSQL. Three roles on the `users.role` column:

- `subscriber` — default, access to own data only
- `moderator` — view users/subscriptions, manage websites and tips
- `administrator` — full access including templates, app settings, and system-wide data

Frontend route guards: `<ProtectedRoute>`, `<AdminRoute>`, `<PublicOnlyRoute>` in `client/src/App.tsx`.

## Payments

- **Stripe Subscriptions**: tiers `basic` / `essential` / `pro`, monthly and yearly. Webhook handler at `POST /api/webhook/stripe`.
- **Stripe Connect**: per-website seller payouts. Separate webhook at `POST /api/webhook/stripe-connect`.
- Price IDs mapped in `server/stripe/` and the `stripePrices` table.

## Internationalization

Translations: `client/src/locales/en.json` and `gr.json`. Use the `useTranslation()` hook. Language auto-detected via `i18next-browser-languagedetector`.
